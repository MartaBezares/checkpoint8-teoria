¿Cuáles son las diferencias entre const, let y var?

Siguiente
pregunta

Vamos a aprender sobre las diferencias entre const, let y var en JavaScript. Estos son tipos de declaraciones de variables que usamos para almacenar datos, pero cada uno tiene características específicas que los hacen adecuados para diferentes situaciones.

Vamos a desglosar cada uno y ver cuándo y por qué usaríamos cada tipo.

  1. Var:

    Es la forma más antigua de declarar variables en JavaScript y tiene algunas características únicas:

    • Ámbito de función: Las variables declaradas con var tienen ámbito de función, lo que significa que son accesibles en toda la función donde se declaran.
    • Hoisting: Las variables declaradas con var son "elevadas" al comienzo de su ámbito. Esto significa que puedes usar la variable antes de su declaración, aunque su valor será indefinido hasta que se inicialice.

  2. Let:

    Es la forma moderna de declarar variables.

    Tiene varias mejoras respecto a var:

    • Ámbito de bloque: Las variables declaradas con let tienen ámbito de bloque, lo que significa que solo son accesibles dentro del bloque {} donde se declaran.
    • No se eleva de la misma manera: Aunque let también se eleva, no se inicializa hasta que la ejecución alcanza su declaración. Intentar acceder a una variable let antes de su declaración resultará en un error.

    Ejemplo:

  3. Const:

    Se introdujo a la vez que let y se usa para declarar variables que no se pueden reasignar (constantes). Sin embargo, no significa que el valor almacenado sea inmutable, solo que la referencia a la variable no puede cambiar.

    Sus características son:

    • Ámbito de bloque: Igual que let, const tiene ámbito de bloque.
    • Debe ser inicializada: Una variable const debe ser inicializada al momento de su declaración y no puede ser reasignada.

    En resumen:

  1. Var:
    • Ámbito: Función.
    • Hoisting: Sí, pero inicializado como indefinido.
    • Reasignable: Sí.
    • ¿Cuándo usar cada uno?: Generalmente se desaconseja su uso en favor de let y const debido a sus problemas de ámbito y hoisting.
  2. Let:
    • Ámbito: Bloque.
    • Hoisting: Sí, pero no se puede acceder antes de la declaración.
    • Reasignable: Sí.
    • ¿Cuándo usar cada uno?: Usa let para variables que necesitas reasignar pero que deben tener ámbito de bloque.
  3. Const:
    • Ámbito: Bloque.
    • Hoisting: Sí, pero no se puede acceder antes de la declaración.
    • Reasignable: No, la referencia no puede cambiar. Sin embargo, puedes modificar los contenidos de objetos y arrays.
    • ¿Cuándo usar cada uno?: Usa const para variables que no deben ser reasignadas. Es una buena práctica preferir const a menos que sepas que necesitas reasignar la variable.
